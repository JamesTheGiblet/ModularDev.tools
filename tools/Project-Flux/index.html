âš¡ The Manifesto: Core Principles

We believe software should be transparent, modifiable, and truly owned by its users. Project Flux is a testament to this belief, built on a philosophy of ultimate flexibility and immediate feedback.

  • Dependency-Injected Systems: Core functionalities like the `WeaponSystem`, `CharacterSystem`, and `RulesSystem` are modular and hot-swappable.
  • No Build Step: Forget compilers and complex environments. Edit, save, refresh. Your changes are instantly reflected in the game.
  • No Licensing: This project is in the Public Domain (or uses a highly permissive MIT License). It's truly yours to use, modify, and redistribute.
  • No Servers: The engine is designed to run independently and can be hosted anywhereâ€”from your local machine to IPFS.
  • No Limits: If you can code it in JavaScript, you can mod it. The game's design explicitly exposes its internal workings for maximum creative freedom.
